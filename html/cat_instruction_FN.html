Naming convention of output files

Images:

Segmented Images: #p[0123]* [m[0]w]p[0123]*[_affine]
Local Intensity Normalized: #m* [w]m*
Atlas Maps: #a* [mw]a1*
Preprocessing Changes Map: #pc* [mw]pc*
Tissue Expectation Map: #te* [mw]te*

* filename
# image space prefix

Image space prefix:
m modulated
m0 modulated non-linear only
w warped (spatially normalized using DARTEL)

Image space extension:
_affine affine registered only

Image data prefix:
p segmented
0 PVE label
1 GM
2 WM
3 CSF

Surfaces:

SURF.TYPE.*

SURF left, right hemisphere [ lh | rh ]
TYPE surface data file [ central | sphere | thickness | logsulc | ... ]
central - coordinates and faces of the central surface
sphere - coordinates and faces of the spherical projection of the central surface
sphere.reg - coordinates and faces of the sphere after spherical registration
thickness - thickness values of the surface
logsulc - log10-transformed values of sulcul depth based on the euclidian distance between the central surface and its convex hull
gyrification - gyrification values based on absolute mean curvature
fractaldimension - fractal dimension values (cortical complexity)

Additional data:

Extracted data, global morphometric measure and image quality measures are stored in the vbm_*.xml file.
You can use the vbm_io_xml function to read data from xml-files.
ROI data is also saved as csv-file vbmROI[SPACE]_[ATLAS]_*.csv.
