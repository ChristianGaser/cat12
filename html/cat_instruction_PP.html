Preprocessing

CAT12 preprocessing comprehends methods to reduce image interference, classify the brain and its major tissue classes and normalize the individual anatomy. Furthermore, it allows the creation of the central surface with cortical thickness and folding measures. The theoretical background can be found in the Methods section.

  • Options for initial SPM12 affine registration

    Although CAT12 relies on SPM12 preprocessing it uses an independent segmentation approach and the SPM12 segmentation is only used as starting estimate. Thus, most parameters were set to SPM default and removed from the GUI. Modification should be made only by expert users via cat_defaults file. Please choose the SPM "Tissue Probability Map" and "Affine Registration" for your data if necessary.

  • Extended options for CAT12 segmentation

    CAT12 uses adaptive denoising methods to allow an reliable segmentation for different levels of noise.

    For the denoising, the Local Adaptive Segmentation, the Skull-Stripping, and the Cleanup function a strength parameter allows simple modifications if the default segmentation does not fit to your requirements. Setting the strength to 0 will deactivate the respective method. Values smaller than the default value of 0.5 will run the functions with less strength, whereas higher values (maximum=1) will increase the strength.

    For children data you may need a customized Dartel template. Be aware of problems with the ROI analysis, because the CAT12 atlas maps are optimized for CAT12 IXI555 template!

    CAT12 also allows the creation of the central cortical surface and the estimation of thickness and several folding parameters. Surface processing will take around 2-3 times longer as segmentation only preprocessing (around 40 min for a standard dataset on a computer of the year 2016).

  • Writing options

    CAT12 allows to write the results of the preprocessing in different spaces - native/subject space, normalized space, modulated normalized space, and DARTEL export. For most CAT analysis the modulated normalized space is the best choice. DARTEL export is only required for DARTEL template generation.

    The following maps can be saved:

    • GM, WM, and CSF class maps (p1, p2, and p3 maps)

      The PVE tissue map for each class, similar to the SPM tissue maps c1, c2, and c3. However, the main difference is that PVE maps provide a measure about the volume fraction of a tissue type. A value of 0.7 in the PVE GM map means that this voxel contains 70% GM and 30% remaining tissue (either WM or CSF depending on the neighbouring voxels).

    • PVE label map (p0 maps)

      A combination of the tissue maps with background=0, CSF=1, GM=2, and WM=3. A value of 2.41 means that the voxel contains of 41% WM and 59% GM.

    • Bias corrected (m maps)

      Before segmentation CAT12 preprocessing removes image interferences such as noise and inhomogeneities from the images. Furthermore, global and local tissue intensities were normalized (background=0, CSF=1/3, GM=2/3, and WM=1) to allow CAT analysis. The optimized T1-image contains more local information, such as small changes in the WM.

    • Jacobian determinant

      Describes the local volume changes and can be used in a pure deformation based morphometry (DBM) design.

    • Deformation fields

      These fields contain the information to map images from native to MNI space (forward), or from MNI to native space (backward). Use Normalise write to apply deformations to a subject without modulation (e.g. for fMRI data). Here you can also change the bounding box and voxel size. If you want to apply deformations to VBM data (e.g. segmentations) you can use the functions Apply deformations (many images) or Apply deformations (many subjects) depending on whether you have many subjects or many images to apply the deformations.

    The resulting structure of the filenames can be found here.

  • Expert options

    Further options and parameter such as the SPM12 parameters used for preprocessing are available for expert users by setting the option "cat12.extopts.expertgui" in cat_defaults.m to "1".
