CAT error management

To avoid preprocessing stops in large datasets by single errors, CAT allows to catch preprocessing errors of on subject (cat.extopts.ignoreErrors in cat_defaults). It creates an error directory "err" within the folder of the file and creates a error specific subdirectory "FUNCTION.LINE.IDENTIFIER" with the information where the LAST error occurred and its identification. The CAT XML file with preprocessing parameters, error and system information are written in this directory. Furthermore, the CAT report (if exist) is copied and a symbolic link to the original file is created (only MAC/Linux) to check the original image.

Although, the LAST error terminates the processing it is often not the major problem. E.g. an error in the cat_vol_morph for an empty image, can occur in the preprocessing function cat_main, but is caused much early by a bad affine registration, failed corrections or miss classifications. Therefore, the XML file is essential to identify the real error.

Failed preprocessing is often related to a failed affine registration that was caused by a bad image orientation. So if your preprocessing failed, please checkt first the image orientation, correct it (if is was bad) and try again. If also this failed, you can sending us the XML-file (and the original image) to help us to identify the problem and correct the error.
