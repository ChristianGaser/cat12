CAT error management

If a preprocessing error occurs an error directory "err" with error specific subdirectories is created within the subject directory. In this directory a copy of the error report PDF and CAT XML of this subject will be created. However, also in a complete preprocessing there might occur problems that can be identified by the standard report PDF. You can find both files "catreport_*.pdf" in the main or report directory of each subject.

Failed preprocessing is often related to failed affine registration that was caused by a bad image orientation. In the CAT report of a failed preprocessing the crosshair is placed at the AC in the input image. Thus, if your preprocessing fails, please check first the image orientation in the CAT error report or the SPM display function and correct it using the "Set Origin" function and try the preprocessing again.

If this also fails, you can use the send error mail function to select the error or processing directories. The function will search for all XML and PDF files that include error messages, will zip them and create a standard email that is opened in you email client. Here, you have to attach the zipped file by yourself and may enter additional information.

To avoid break of preprocessing of large datasets by single errors, CAT allows to catch preprocessing errors of each subject (cat.extopts.ignoreErrors in cat_defaults).
