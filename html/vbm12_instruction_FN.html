Naming convention of output files

Images:

Segmented Images: #p[0123]* [m[0]w[r]]p[0123]*[_affine]
Local Intensity Normalized: #m* [w[r]]m*
Atlas Maps: #a* [mw[r]]a1*
Preprocessing Changes Map: #pc* [mw[r]]pc*
Tissue Expectation Map: #te* [mw[r]]te*

* filename
# image space prefix

Image space prefix:
m modulated
m0 modulated non-linear only
w warped (spatially normalized)
r DARTEL warped
_affine affine registered only

Image data prefix:
p segmented
0 PVE label
1 GM
2 WM
3 CSF

Surface data prefix:

SURF.TYPE.[nofix.]*

SURF left, right hemisphere [ lh | rh ]
TYPE surface data file [ central | sphere | thickness | curvature | ... ]
central - coordinates and faces of the central surface
sphere - coordinates and faces of the spherical projection of the central surface
thickness - thickness values of the surface
curvature - curvature value of the surface
nofix raw surface with uncorrected topology
reg spherically registered surface

Further data:

Extracted data, global morphometric measure and image quality measures were stored in the vbm_*.xml file.
You can use the vbm_io_xml function to read data from xml-files.
ROI data is also saved as csv-file vbmROI[SPACE]_[ATLAS]_*.csv.
