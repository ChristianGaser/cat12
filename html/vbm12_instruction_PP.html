Preprocessing

VBM12 preprocessing comprehends methods to reduce image interference, classify the brain and its major tissue classes and normalize the individual anatomy. Furthermore, it allows the creation of the central surface with cortical thickness and folding measures. The theoretical background can be found in the Methods section.

VBM12 also allows processing of other apes and monkeys. First, you have to download the animal templates and default parameter. Next you have to change the default file for your primate.

  • Estimation options

    Although VBM12 uses SPM12 preprocessing it uses an independent segmentation approach and the SPM12 segmentation is only used as starting estimate. Thus, most parameters were set to SPM default and removed from the GUI. Modification should be made only by expert users via cg_vbm_defaults file. Please choose the SPM "Tissue Probability Map" and "Affine Registration" for your data.

  • Extended options

    VBM12 uses adaptive denoising methods to allow a exact and stable segmentation for different levels of noise.

    For the Local Adaptive Segmentation, the Skull-Stripping, and of the Cleanup function a strength parameter allows simple modifications, if the default segmentation does not fit for your requirements. Setting the strength to 0 will deactivate the method. Values smaller than the default value of 0.5 will run the functions with less power, whereas higher values (maximum=1) will increase the power.

    For very young children and diseases with strong anatomical properties you may require a special Dartel template. Be aware of problems with the ROI analysis, because the VBM12 atlas maps are optimized for VBM12 IXI555 template!

    For regional analysis the Hammers atlas in IXI555 template space allows extraction of values such as thickness. Results were stored in the vbm_*.XML and a csv-file vbmROI*.csv files.

    VBM12 also allows the creation of the central cortical surface and the estimation of thickness and folding parameters. Surface processing will take around 2 times longer as segmentation only preprocessing (around 1 hour for a standard dataset on a computer of the year 2013).

  • Writing options

    VBM12 allows to write the results of the preprocessing in different spaces - native/subject space, normalized space, modulated normalized space, and DARTEL export. For most VBM analysis the modulated normalized or normalized space are the best choice. DARTEL export is only required for special analysis (BrainAGE) or for DARTEL template generation.

    There are the following map types:

    • GM, WM, and CSF class maps (p1, p2, and p3 maps)

      The PVE tissue map for each class, similar to the SPM tissue maps c1, c2, and c3. However, the main difference is that PVE maps provide a measure about the volume fraction of a tissue type. A value of 0.7 in the PVE GM map means that this voxel contains 70% GM and 30% remaining tissue (either WM or CSF depending on the neighbouring voxels).

    • PVE label map (p0 maps)

      A combination of the tissue maps with background=0, CSF=1, GM=2, and WM=3. A value of 2.41 means that the voxel contains 41% WM and 59% GM.

    • Bias corrected (m maps)

      Before segmentation VBM12 preprocessing removes image interferences like noise and inhomogeneity from the images. Furthermore, global and local tissue intensities were normalized (background=0, CSF=1/3, GM=2/3, and WM=1) to allow VBM analysis. The optimized T1-image contain more informations, such as small changes in the WM. On the other hand, they were also stronger affected by the scan protocol and uncorrected image interferences.

    • Jacobian determinant

      Describes the local volume changes and can be used in a pure deformation based morphometry (DBM) design.

    • Deformation fields

      These fields contain the information to map images from native to MNI space (forward), or from MNI to native space (backward). Use the Deformation Utility to apply deformations to a subject.

    Further options and parameter such as the SPM12 parameter used for preprocessing are available for expert users via cg_vbm_defaults file.
